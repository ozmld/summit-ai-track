## Слайд 5. sklearn-API — магия одного интерфейса

**Любая модель** в sklearn выглядит так:

```python
from sklearn.<семейство> import <Модель>

model = Модель()
model.fit(X_train, y_train)
model.score(X_test, y_test)
model.predict(X_new)
```

Меняется **только** название класса. Код — тот же.

## Слайд 6. Живой пример — три модели одним кодом

```python
from sklearn.linear_model import LogisticRegression
from sklearn.neighbors import KNeighborsClassifier
from sklearn.ensemble import RandomForestClassifier

for name, M in [
    ('LogReg', LogisticRegression(max_iter=1000)),
    ('KNN',    KNeighborsClassifier(n_neighbors=5)),
    ('Forest', RandomForestClassifier()),
]:
    M.fit(X_train, y_train)
    print(name, M.score(X_test, y_test))
```

Смотрим что меняется → только имя модели. Метод `fit / score` везде один.

## Слайд 7. Ключевое правило — train ≠ test

**train** — на чём учим.
**test** — на чём проверяем.

Если дать один и тот же набор — модель «запомнит», а не обучится.

```python
from sklearn.model_selection import train_test_split
X_train, X_test, y_train, y_test = train_test_split(
    X, y, test_size=0.3, stratify=y, random_state=42
)
```

## Слайд 8. Accuracy — и почему её мало

**Accuracy** = доля правильных ответов.

Когда врёт: если 95% объектов одного класса, модель «всегда отвечай 1» даст 95% и выглядит классно. На деле — бесполезна.

## Слайд 9. Precision и recall — два взгляда

| метрика | вопрос |
|---|---|
| **Precision** | когда я сказал «да», я был прав? |
| **Recall** | я не пропустил важное? |

**Медицинский пример:** тест на болезнь.

- Высокий precision, низкий recall — редко поднимаем ложную тревогу, но пропускаем больных.
- Высокий recall, низкий precision — находим всех больных, но кучу здоровых пугаем.

---

## Слайд 10. F1 — компромисс

$$ F_1 = \frac{2 \cdot \text{precision} \cdot \text{recall}}{\text{precision} + \text{recall}} $$

Когда важны оба — смотрим F1. Это «среднее» precision и recall, но штрафующее за перекос.

## Слайд 11. Confusion matrix — картинка, которая всё объясняет

```python
from sklearn.metrics import ConfusionMatrixDisplay
ConfusionMatrixDisplay.from_estimator(model, X_test, y_test)
```

На диагонали — правильные. Вне диагонали — ошибки. Смотрим, **какие классы путаем**.

Это первый график, который я строю на любой классификации.

## Слайд 12. Регрессия за 3 минуты

**MSE** — среднеквадратичная ошибка. Штрафует большие промахи квадратично.
**MAE** — средняя абсолютная ошибка. Честнее по смыслу.
**R²** — «насколько лучше, чем просто среднее». 1 — идеал, 0 — не лучше среднего, <0 — хуже.

```python
from sklearn.metrics import mean_squared_error, mean_absolute_error, r2_score
```

## Слайд 15. Что сказать классу

1. `sklearn` — это «гайка одного ключа». Выучил `fit / predict / score` — знаешь интерфейс 100 моделей.
2. Одной метрики не хватает. Accuracy врёт на перекошенных данных.
3. Precision — «не ошибаюсь ли, когда говорю да». Recall — «не пропускаю ли важное». F1 — компромисс.
4. Если train ≫ test — модель запомнила, а не поняла.
5. Визуал — не украшение. Scatter + confusion matrix за минуту объясняют больше, чем таблица из цифр.
