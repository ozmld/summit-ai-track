# Гайд для учителей по командному проекту

**II Саммит талантов · трек ИИ · Астана, 25–26 апреля 2026**

---

## Коротко

За два дня саммита ваша команда собирает один **олимпиадный модуль по ИИ** — набор материалов, с которым вы можете прийти в класс и провести 1–2 урока подготовки к Всероссийской олимпиаде по искусственному интеллекту.

Всё хранится в одной папке команды на Google Drive. К концу Дня 3 в папке должен лежать готовый модуль, ссылка на который будет опубликована на сайте саммита.

---

## Что такое модуль

Законченный документ из **5 разделов** (0–4):

- **Раздел 0.** Шапка: название, состав команды, тема, тип задачи.
- **Раздел 1.** Условие задачи в формате ВсОШ.
- **Раздел 2.** Решение + ответ.
- **Раздел 3.** Методичка для ученика — как думать, типичные ошибки, подсказки.
- **Раздел 4.** Гайд для учителя — тайминг урока, вопросы для класса, критерии оценивания.

Плюс пять опциональных **бустов** (каждый +5 баллов к итогу): устойчивость к LLM, реальный кейс, серия задач, визуал, интерактив.

Полное ТЗ: <https://summit-ai-track.vercel.app/project>

---

## Где всё лежит

**1. Общая папка саммита на Google Drive.**
В ней вы найдёте папку **«Команды трека ИИ»**, а внутри — папку с названием вашей команды.

**2. В папке команды — три файла:**
- `README.md` — эта же инструкция в сокращённом виде.
- `module-template.md` — шаблон модуля со всеми пятью разделами. **Именно с него начинаем.**
- `resources/` — сюда складывать всё дополнительное (картинки, ноутбуки, скрины диалогов с LLM).

**3. Сайт саммита.** На нём — расписание, лекции, ссылки на ноутбуки и ТЗ проекта.

## Как сделать модуль (пошагово)

### Шаг 1. Создать рабочий документ из шаблона

1. Откройте папку своей команды.
2. Правой кнопкой по `module-template.md` → «Открыть с помощью» → **Google Docs**.
3. Google преобразует файл в Google Doc. Сохраните его как `Модуль — [НАЗВАНИЕ КОМАНДЫ]`.
4. В шапке документа выдайте доступ «Все, у кого есть ссылка» → «Просмотр» (для peer review на Дне 3).

*Альтернатива:* можно работать прямо в Google Docs «с нуля», используя разделы шаблона как структуру.

### Шаг 2. Заполнять постепенно, от пары к паре

| Когда | Что пишем |
|-------|-----------|
| Д2П1 (конец пары и между парами) | Раздел 0 — шапка команды |
| Д2П4 (первая половина) | Раздел 1 — условие задачи |
| Д2П4 (вторая половина) | Раздел 2 — решение и ответ |
| Д3 (первая пара проекта) | Раздел 3 — методичка для ученика |
| Д3 (вторая пара проекта) | Раздел 4 — гайд для учителя |
| Д3 (в процессе, параллельно) | Материалы под взятые бусты (ноутбук, картинки, версия с подвохом для LLM и т.д.) |

Не успеваете раздел? Ничего страшного — догоняете на следующей паре или вечером.

### Шаг 3. Добавлять файлы в `resources/`

- Картинки / схемы — сразу в папку, затем вставляете ссылкой в документ.
- Colab-ноутбук — делаете «Доступ по ссылке», кладёте ссылку в соответствующий раздел документа.
- Скрин диалога с ChatGPT / DeepSeek — картинкой в папку, ссылкой в разбор буста «Л».

### Шаг 4. Финализировать к защите

Последняя пара Дня 3 — peer review. К её началу у вас должно быть:

- Документ полностью заполнен.
- Доступ «по ссылке» — любой учитель может открыть и прочесть.
- В шапке документа стоит галочка напротив взятых бустов.
- Ссылка на документ опубликована через форму на сайте (её кураторы откроют в Д3П1).

## Как будут оценивать

### Рубрикатор (максимум 100 баллов)

| Критерий | Баллы |
|----------|-------|
| Оригинальность задачи | 20 |
| Корректность и полнота решения | 20 |
| Качество методички для ученика | 20 |
| Гайд для учителя | 20 |
| Peer review от коллег | 20 |

### Бусты (до 5 штук, по +5 за каждый)

- **Л** — Устойчивость к LLM: версия задачи с подвохом для ИИ + пример сбитого ответа + объяснение механики (где и почему LLM ломается).
- **К** — Реальный кейс: привязка к новости / данным с источником.
- **С** — Серия: 2–3 связанные задачи возрастающей сложности.
- **В** — Визуал: собственная картинка / схема / график.
- **И** — Интерактив: ноутбук Colab или веб-демо.

### Peer review

Классической презентации не будет. На Д3П2 все учителя заходят на сайт, открывают модули коллег и оценивают по четырём шкалам:

- Полезно мне как учителю (0–5).
- Понятность методички для ученика (0–5).
- Оригинальность задачи (0–5).
- Хочу взять в класс (да / нет).

Жюри параллельно ставит свои оценки. По сумме определяются два приза: **«Выбор учителей»** (по peer review) и **«Приз жюри»**.

## Частые вопросы

**Нам обязательно делать задачу с подвохом для LLM?**
Нет. Это отдельный буст **Л** (+5 баллов) — берёте, только если хотите. Без него модуль на пяти разделах (0–4) считается полным.

**Можно ли взять задачу из учебника / готового задачника?**
Лучше оригинальную. Если берёте существующую — **существенно переделайте** условие (числа, контекст, формат ответа). В Разделе 0 укажите источник.

**Сколько бустов можно взять?**
Хоть все пять. Но каждый должен быть реально реализован, а не только отмечен в шапке. Жюри проверяет.

**Что если команда не успевает?**
Работайте сколько успеете. Минимально — заполненный Раздел 1 (условие) и Раздел 2 (решение). Без них модуль не будет участвовать в peer review.

**Можно ли писать не в Google Docs?**
Можно, но итоговая ссылка должна открываться из браузера без скачивания (Google Docs, Notion с публичной ссылкой, HackMD и т.п.). Всё остальное — только вложения в папку команды.

**Кому задать вопрос во время работы?**
В Д2П4 в зале будут все пять менторов. В остальные пары подходите к Ивану Эйдлину — координатору трека.
